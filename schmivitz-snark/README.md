# schmivitz-snark

A SNARK-based verification system for VOLE-in-the-head proofs generated by schmivitz.

## Overview

schmivitz-snark is an extension of the [schmivitz](../schmivitz) library that provides SNARK-based verification for VOLE-in-the-head zero-knowledge proofs. It allows you to convert proofs generated by schmivitz into Groth16 SNARK proofs, which can be efficiently verified on-chain or in other resource-constrained environments.

The main advantages of using schmivitz-snark include:
- Constant-time verification regardless of circuit complexity
- Significantly reduced verification gas costs for on-chain verification
- Compatibility with existing Ethereum SNARK verifiers
- Automatic generation of Solidity verification contracts

## Features

- Convert schmivitz proofs to SNARK proofs
- Generate and verify SNARK proofs using the Groth16 proving system
- Export verification contracts to Solidity for on-chain verification
- Preserve the security guarantees of the original VOLE-in-the-head proofs

## Installation

Add schmivitz-snark to your Cargo.toml:

```toml
[dependencies]
schmivitz-snark = { path = "../schmivitz-snark" }
schmivitz = { path = "../schmivitz" }
```

## Usage

### Basic Usage

```rust
use schmivitz::proof::Proof;
use schmivitz::vole::insecure::InsecureVole;
use schmivitz_snark::{convert_proof, prove, setup, verify, SnarkKeys};
use rand::thread_rng;

// Generate a proof using schmivitz
let schmivitz_proof = Proof::<InsecureVole>::prove(
    &mut circuit,
    &private_input_path,
    &mut transcript,
    &mut rng,
)?;

// Set up the SNARK proving and verification keys
let rng = &mut thread_rng();
let keys = setup(rng)?;

// Convert the schmivitz proof to a VoleProof
let vole_proof = convert_proof(&schmivitz_proof)?;

// Generate a SNARK proof
let snark_proof = prove(&vole_proof, &keys, rng)?;

// Verify the SNARK proof
let is_valid = verify(&snark_proof, &keys)?;
assert!(is_valid);
```

### Generating a Solidity Verifier

The `setup` function automatically generates a Solidity verifier contract in the `solidity_output` directory:

```rust
let keys = setup(rng)?;
// A Solidity verifier is generated at solidity_output/vole_verifier.sol
```

You can then deploy this contract to Ethereum or other EVM-compatible blockchains to verify proofs on-chain.

## API Reference

### Main Functions

#### `setup`

```rust
pub fn setup<R: Rng + CryptoRng>(rng: &mut R) -> Result<SnarkKeys>
```

Generates the proving and verification keys for the SNARK system. This function also outputs a Solidity verifier contract to the `solidity_output` directory.

#### `convert_proof`

```rust
pub fn convert_proof(schmivitz_proof: &Proof<InsecureVole>) -> Result<VoleProof>
```

Converts a proof generated by schmivitz into a VoleProof that can be used with the SNARK system.

#### `prove`

```rust
pub fn prove<R: Rng + CryptoRng>(
    vole_proof: &VoleProof,
    keys: &SnarkKeys,
    rng: &mut R,
) -> Result<SnarkProof>
```

Generates a SNARK proof from a VoleProof using the provided proving keys.

#### `verify`

```rust
pub fn verify(snark_proof: &SnarkProof, keys: &SnarkKeys) -> Result<bool>
```

Verifies a SNARK proof using the provided verification keys.

### Data Structures

#### `VoleProof`

```rust
pub struct VoleProof {
    pub vole_challenge: F128b,
    pub witness_commitment: Vec<F64b>,
    pub witness_challenges: Vec<F128b>,
    pub degree_0_commitment: F128b,
    pub degree_1_commitment: F128b,
    pub decommitment_challenge: F128b,
    pub partial_decommitment: PartialDecommitment,
}
```

Represents a VOLE-in-the-head proof in a format compatible with the SNARK system.

#### `SnarkKeys`

```rust
pub struct SnarkKeys {
    pub proving_key: ProvingKey<Bn254>,
    pub verification_key: VerifyingKey<Bn254>,
}
```

Contains the proving and verification keys for the SNARK system.

#### `SnarkProof`

```rust
pub struct SnarkProof {
    pub proof: Groth16Proof<Bn254>,
    pub inputs: Vec<Bn254Fr>,
}
```

Represents a SNARK proof that can be efficiently verified.

## Implementation Details

schmivitz-snark implements a circuit that verifies the validity of a VOLE-in-the-head proof. The circuit includes:

1. Computation of masked witnesses
2. Circuit traversal to compute the validation aggregate
3. Verification of the final constraint equation

The implementation uses the arkworks library for the underlying SNARK functionality and the Groth16 proving system.

## Limitations

- Currently only supports the InsecureVole implementation from schmivitz
- The circuit structure must be available during verification
- The Solidity verifier requires a compatible version of Solidity (>=0.8.0)
